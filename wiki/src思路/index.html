SRC思路

1.信息收集

1.src企业的ip子域名收集

1.通过fofa，钟馗之眼，shodan等搜集子域名。

2.通过oneforall或subDomainsBrute来收集

​ subDomainsBrute：

​ –version 显示版本号

​ -h 帮助

​ -f FILE 暴力猜解的字典，默认使用自带的subnames.txt

​ –full 全扫描，使用subnames_full.txt

​ -i 忽略不想采集的ip地址

​ -t 扫描的线程数，默认200

​ -p 扫描的进程数，默认6

​ -o 输出文件的名称

​ eg.创建20个线程访问报读，将结果保存到baidu.txt

​ python subDomainsBrute.py -t 20 http://baidu.com -o baidu.txt

2.查看目标的资产

可以通过小蓝本搜索，可以得到备案网站，APP，小程序，公众号，甚至地址等等。

小蓝本：https://www.xiaolanben.com/

可以利用goboy进行资产的绘测。

3.利用google进行在线网站信息收集

比较常用的Google语法：

inurl:login|admin|manage|member|admin_login|login_admin|system|login|user|main|cms 寻找后台登录

site:域名 intext:管理|后台|登陆|用户名|密码|验证码|系统|帐号|admin|login|sys|managetem|password|username 同上

inurl:aspx|jsp|php|asp 寻找注入点

inurl:file|load|editor|Files 寻找上传点

inurl:ewebeditor|editor|uploadfile|eweb|edit 寻找web编辑器

filetype:mdb|asp|# 寻找数据库

site:xxx.com intext:@xxx.com／intext:@xxx.com 寻找邮箱

4.使用github搜索企业泄露的信息源码

GitHub语法：

1.q+关键字

2.in:file,path 在文件名和路径中搜索包含关键字信息

3.repo:项目名 在指定项目中搜索包含敏感信息的文件

或者使用工具进行搜索

5.敏感目录，端口扫描，cms识别

使用御剑，k8等工具进行敏感目录扫描

使用nmap进行端口扫描，常用语法 nmap -sS -n -A 域名

cms识别方法

国内:http://whatweb.bugscaner.com/

​ http://finger.tidesec.net/

​ https://www.yunsee.cn/3

国外:https://whatcms.org/

​ https://cmsdetect.com/

​ https://itrack.ru/whatcms/

2.寻找与数据库有信息交互的地方

根据前面得到的信息，寻找与数据库有交互的地方。

比如后台，登录页面，查询信息，页面参数等地方。

3.漏洞扫描

sql注入可以使用sqlmap进行注入

xss漏洞可以利用Xsstrike工具以及Burp suite插件之xss Validator

漏洞的扫描可以利用burpsuit+xary和awvs+xary

4.爆破

爆破使用burpsuite抓包爆破

验证码绕过爆破可以利用tesserocr+burpsuite

5.业务逻辑漏洞测试

有登录后的垂直越权，水平越权，及未授权访问

验证码的重复验证导致短信轰炸

支付漏洞，抓包改金额，活动褥羊毛

密码找回模块和登录模块的问题
